SQL basics

  1. sqlite3 test.db shows the version of sql on the computer
  2. ls -l shows that it created a db (this is command line code)
  3. sqlite3 this gets you in to the sqlite prompt
  4. CREATE TABLE person (
    id INTEGER PRIMARY KEY,
    first_name TEXT,
    last_name TEXT,
    age INTEGER
    );
     This creates the schema for a database. It can be made in sqlite prompt or in .sql file
  5. .quitthis gets you out of the sqlite prompt
  6. ;if you get stuck in prompt this is probably what's missing
  7. .schemathis lets you look at the schema. The database must be loaded with sqlite3 ex2.db
  8. INSERT INTO person (id, first_name, last_name, age) VALUES (0, "Zed", "Shaw", 37); this is an .sql file that will insert info into a database. first parens is column, second is row.
  9. -echothis command prints out what the computer is doing. the command it is doing is after echo
  10. -column The full line is sqlite3 -column ex3.db < ex6.sql. This access the column from the database requested using the select method written in the sql file.
  11. -header seems to do the same thing? as column
  12. cat << EOF > ~/.sqliterc .headers on .mode column EOFThis is why header and column look the same because this was set as the default.
  13. SELECT pet.id, pet.name, pet.age, pet.dead FROM pet, person_pet, person WHERE pet.id = person_pet.pet_id AND person_pet.person_id = person.id AND person.first_name = "Zed"; This is a query joining person and pet tables through the person_pet table using AND instead of join.
  14. DELETE FROM pet WHERE dead = 1;This is the syntax for deleting a row that has column(dead) equal to 1,
  15. SELECTusually means give me this row
  16. FROMmeans from this table
  17. WHEREmeans here is the beginning of a condition that needs to be met for the selected row
  18. INthis is where it can start to get convoluted. It can start a subquery or it can mean in a group that I give you surrounded by (). Like IN('France','Germany'). this is IN the rows named France and Germany.
  19. ''values from columns need to be in quotes if they are text or you'll get errors.
  20. LIKEpermits pattern matching with % that can go before or after a letter to see if it starts or end with the letter. Like 'D%' asks if it starts with D.
  21. BETWEENallows for range checking. BETWEEN 1 AND 100.
  22. ANDused like a normal boolean. Example above.
  23. UPDATE person SET first_name = "Hilarious Guy" WHERE first_name = "Joe";This changes the original value of first name from "Joe" to "Hilarious Guy".
  24. SELECT * FROM game JOIN goal ON (id=matchid)
    25. This code selects all columns from two tables. Tables are named game and goal and it is joining on an id number that is the same for both. The id and match id are what creates the relation between these two table.
  25. SELECT name FROM world WHERE population > (SELECT population FROM world WHERE name='Russia') This is a nested SELECT statement it's selecting the name of a country in the world that has a population greater than that population of Russia.
  26. SUMAdds all the numbers of a column. Example:SELECT SUM(population) FROM world; This gives the sum of the column population from the table world. Make sure there are no spaces between SUM and ().
  27. DISTINCTlists the distinct values from a column. Example: SELECT DISTINCT continent FROM world;
  28. COUNTIt counts the number of rows for a given constraint. Example:SELECT COUNT(name) FROM world WHERE area > 1000000; this counts how many names are in the column name from the world table that have area over 10 million.
  29. GROUP BYIt finds the distinct value of that column and then lists it. Example: SELECT continent, COUNT(name) FROM world GROUP BY continent; This selects the column and number of countries from the world and groups it together by the distinct values in the continent column which are the continent names.
Datatypes and Storage Class

NULL. The value is a NULL value.

INTEGER. The value is a signed integer, stored in 1, 2, 3, 4, 6, or 8 bytes depending on the magnitude of the value. REAL. The value is a floating point value, stored as an 8-byte IEEE floating point number. TEXT. The value is a text string, stored using the database encoding (UTF-8, UTF-16BE or UTF-16LE). BLOB. The value is a blob of data, stored exactly as it was input.
